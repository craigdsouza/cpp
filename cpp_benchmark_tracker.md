# C++ Proficiency Benchmark Tracker

**Goal:** Track C++ progress across competitive programming (Codeforces), interview-style problems (LeetCode), and systems/NVIDIA-aligned skills (custom rubric).

**Cadence:** Every ~4 weeks. First benchmark on Day 14 (~April 9, 2026).

## Systems C++ Rubric — Scoring Key

Each skill is scored **0–3**:

| Score | Meaning |
|-------|---------|
| 0 | Not yet covered / no exposure |
| 1 | Exposed — can read and follow it |
| 2 | Can write it from scratch with some thought |
| 3 | Fluent — would pass a code review |

### Rubric Categories (unlock as curriculum progresses)

| Category | Skills Covered | Unlocked At |
|----------|---------------|-------------|
| **A. Core Syntax** | Types, control flow, functions, const, range-for | Day 1 ✅ |
| **B. References & Pointers** | T&, const T&, T*, nullptr, dangling refs | Day 2 ✅ |
| **C. Basic OOP** | struct, class, constructors, access modifiers, member functions | Day 3 |
| **D. STL Containers** | vector, map, unordered_map, set, pair | Day 5 |
| **E. STL Algorithms** | sort, find, transform, accumulate, ranges | Day 7 |
| **F. Memory Management** | new/delete, stack vs heap, unique_ptr, shared_ptr, RAII | Day 8 |
| **G. Modern C++ (C++17)** | auto, structured bindings, lambdas, move semantics | Day 10 |
| **H. Concurrency** | std::thread, mutex, lock_guard, atomics | Day 16 |
| **I. Systems Patterns** | Error handling, design patterns, const correctness, interfaces | Day 20 |
| **J. AV/NVIDIA Aligned** | Real-time constraints, protobuf/gRPC basics, sensor pipeline concepts | Day 30+ |
